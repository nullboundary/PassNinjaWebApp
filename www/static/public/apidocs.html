Create A New Pass Template

Creates a new empty pass template based on one of the five passbook pass types.

  • URL

    /api/v1/passes

  • Method:

    POST

  • URL Params

    None

  • Data Params

    Required:

     {
      "name": "deep discount coupon",
      "status": "1",

        "description": "A super sale from Deep Discounts Co.",
        "organizationName": "Deep Discounts Co."

Get Pass Link

Feteches a url that can be used to download passbook files generated from the users matching pass template. The return pass url is used to share the pass online.

  • URL

    /api/v1/passes/{pass-id}/link

  • Method:

    GET

  • Arguments

    Required:

    pass-id - The ID of the pass template. The pass ID is a base64 encoded hash.

  • Data Params

    None

  • Success Response:

    • Code: 200 OK
      Content: {"name": hotelpass, "url": https://pass.ninja/pass/1/passes/hotelpass-1329231861}

  • Error Response:

    • Code: 401 UNAUTHORIZED
      Content: { "code": 401, "error" : "Unauthorized" }

      OR

    • Code: 404 NOT FOUND
      Content: { "code": 404, "error" : "Pass matching ID not found" }

      OR

    • Code: 403 FORBIDDEN
      Content: { "code": 403, "error" : "Requested pass is incomplete" }

  • Sample Call:
curl https://pass.ninja/api/v1/passes/wwOoZP6On9nvRPOSGFq9M6ghUO3ncbH5nnzLlWaMYso=/link \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <Access-Token>" \
  -X GET

  • Notes:

    None

Get Mutate List

Fetches a list of keys that can be used to mutate or change values in the pass template for each individual generated pass. The returned list can be used to make a key value pair json object which is passed to the mutatePass method.

  • URL

    /api/v1/passes/{pass-id}/mutate

  • Method:

    GET

  • Arguments

    Required:

    pass-id - The ID of the pass template. The pass ID is a base64 encoded hash.

  • Data Params

    None

  • Success Response:

    • Code: 200 OK
      Content: {"name": hotelpass, "mutatelist": ["itemKey1","anotherKey","moreKeyNames"]}

  • Error Response:

    • Code: 401 UNAUTHORIZED
      Content: { "code": 401, "error" : "Unauthorized" }

      OR

    • Code: 404 NOT FOUND
      Content: { "code": 404, "error" : "Pass matching ID not found" }

      OR

    • Code: 403 FORBIDDEN
      Content: { "code": 400, "error" : "Requested pass is incomplete" }

  • Sample Call:
curl https://pass.ninja/api/v1/passes/wwOoZP6On9nvRPOSGFq9M6ghUO3ncbH5nnzLlWaMYso=/mutate \
  -H "Accept: application/json" \
  -H "Authorization: Bearer <Access-Token>" \
  -X GET

  • Notes:

    None

Mutate Pass Template

This method allows you to modify a passes value fields with unique data and returns a link to the unique generated pass.

  • URL

    /api/v1/passes/{pass-id}/mutate

  • Method:

    PATCH

  • URL Params

    Required:

    pass-id - The ID of the pass template. The pass ID is a base64 encoded hash.

  • Data Params

    Required:

    Calling the method GET /api/v1/passes/{id}/mutate will return a JSON doc listing the mutate-able keys that can be used for constructing this methods request.

    {
      "primary1": "Mr. Roboto",
      "aux1": 2,
      "aux4": "Kyoto",
      "second3": 123
    }

  • Success Response:

    • Code: 200 OK
      Content: {"name": "hotelpass", "url": "https://pass.ninja/pass/1/passes/hotelpass-1329231861"}

  • Error Response:

    • Code: 401 UNAUTHORIZED
      Content: { "code": 401, "error" : "Unauthorized" }

    OR

    • Code: 404 NOT FOUND
      Content: { "code": 404, "error" : "Pass not found" }

    OR

    • Code: 400 BAD REQUEST
      Content: { "code": 400, "error" : "The submitted pass JSON structure is malformed" }

    OR

    • Code: 400 BAD REQUEST
      Content: { "code": 400, "error" : "Mutate field key not found: aux9" }

    OR

    • Code: 403 FORBIDDEN
      Content: { "code": 403, "error" : "Requested pass is incomplete or not mutatable" }

    OR

    • Code: 409 CONFLICT
      Content: { "code": 409, "error" : "A conflict has occurred creating the pass" }

  • Sample Call:

    curl https://pass.ninja/api/v1/passes/wwOoZP6On9nvRPOSGFq9M6ghUO3ncbH5nnzLlWaMYso=/mutate \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer <Access-Token>" \
      -X PATCH \
      -d "{
        'primary1': 'Mr. Roboto',
        'aux1': 2,
        'aux4': 'Kyoto',
        'second3': 123
      }"

  • Notes:

    None

  • Code: 401 UNAUTHORIZED
    Content: { "code": 401, "error" : "Unauthorized" }

  • Code: 400 BAD REQUEST
